梯度下降解密：可视化指南Gradient Descent Demystified: A Visual Guide

2026-06-15

梯度下降是现代优化的主力工具。无论是训练神经网络还是拟合逻辑回归，你几乎一定在使用它的某种变体。本文带你走一遍核心思想，附代码。

什么是梯度下降？

给定一个函数 f(x)，我们希望找到使其最小化的 x。梯度 ∇f 指向最陡上升方向，因此我们沿相反方向移动：

x_{k+1} = x_k - α ∇f(x_k)

其中 α 是步长（学习率）。太小则缓慢爬行；太大则过冲。

精确线搜索的最速下降法

最简单的变体：每一步找到在下降方向上恰好最小化 f 的 α。对于二次函数，存在闭式解。实际中我们使用：

def exact_line_search(f, grad, x, d):
    """沿方向 d 寻找最优步长"""
    alpha = 1.0
    c = 1e-4
    while f(x + alpha * d) > f(x) + c * alpha * grad(x).dot(d):
        alpha *= 0.5
    return alpha

非精确线搜索：Armijo 与 Wolfe 条件

精确线搜索代价高昂。实践中我们使用 Armijo 条件（充分下降）：

f(x + α d) ≤ f(x) + c·α · ∇f(x)&supT; d

结合曲率条件（Wolfe），这在保证收敛的同时快得多。

牛顿法

牛顿法利用二阶信息——海森矩阵 H——同时选择方向和步长：

x_{k+1} = x_k - H(x_k)¹ ∇f(x_k)

在最优解附近具有二次收敛性，但计算 H 为 O(n²)，求逆为 O(n³)。拟牛顿法（BFGS, DFP）通过迭代逼近逆海森矩阵，总代价 O(n²)。

信赖域方法

信赖域方法不先选择方向，而是先划出一个区域，在该区域内我们信任二次近似。子问题：在半径约束下最小化模型。对非凸问题尤为稳健。

方法选型指南

方法每次迭代代价收敛性适用场景

牛顿法O(n³)二次小规模 n，需高精度

实验结果

在修正 Rosenbrock 函数（经典的香蕉形测试问题）上的对比如下：

  • 2D 修正 Rosenbrock：牛顿法约 15 次迭代收敛。精确线搜索最速下降需 500+ 次，Wolfe 线搜索需 80+ 次。
  • 4D 扩展：差距进一步拉大。牛顿/BFGS 在 50 次迭代内完成；最速下降超过 1000 次。

Gradient descent is the workhorse of modern optimization. Whether you're training a neural network or fitting a logistic regression, you're almost certainly using some variant of it. This post walks through the key ideas, with code.

What is Gradient Descent?

Given a function f(x), we want to find the x that minimizes it. The gradient ∇f points in the direction of steepest ascent, so we move in the opposite direction:

x_{k+1} = x_k - α ∇f(x_k)

where α is the step size (learning rate). Too small and we crawl; too large and we overshoot.

Steepest Descent with Exact Line Search

The simplest variant: at each step, find the α that exactly minimizes f along the descent direction. For quadratic functions, there's a closed form. In practice, we use:

def exact_line_search(f, grad, x, d):
    """Find optimal step size along direction d"""
    alpha = 1.0
    c = 1e-4
    while f(x + alpha * d) > f(x) + c * alpha * grad(x).dot(d):
        alpha *= 0.5
    return alpha

Inexact Line Search: Armijo & Wolfe Conditions

Exact line search is expensive. In practice we use the Armijo condition (sufficient decrease):

f(x + α d) ≤ f(x) + c·α · ∇f(x)&supT; d

Combined with the curvature condition (Wolfe), this guarantees convergence while being much faster.

Newton's Method

Newton's method uses second-order information — the Hessian H — to pick both direction and step size:

x_{k+1} = x_k - H(x_k)¹ ∇f(x_k)

Quadratic convergence near the optimum, but computing H is O(n²) and inverting it is O(n³). Quasi-Newton methods (BFGS, DFP) approximate the inverse Hessian iteratively for O(n²) total.

Trust Region Methods

Instead of picking a direction first, trust region methods pick a region around the current point where we trust the quadratic approximation. The subproblem: minimize the model subject to a radius constraint. Particularly robust for non-convex problems.
